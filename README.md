## JobFinder — мобильное приложение для поиска работы

**JobFinder** — учебное мобильное приложение, в котором соискатель может просматривать карточку вакансии и откликаться на неё, выбирая одно из своих резюме. Проект выполнен в рамках командного задания.

### О проекте

Цель проекта — продемонстрировать типичный пользовательский сценарий в приложении для поиска работы: просмотр деталей вакансии и отправка отклика с выбранным резюме.

### Функциональность текущей версии

- просмотр экрана деталей вакансии;
- выбор резюме из списка моковых данных;
- попытка отклика на вакансию;
- отображение предупреждения при выборе неподходящего резюме;
- имитация проверки соответствия навыков через отдельный use case.

### Технологический стек

- **TypeScript**
- **React Native**
- **Expo**
- **Expo Router**
- **React 18.2**
- стандартные компоненты `react-native` (`View`, `Text`, `ScrollView`, `TouchableOpacity`, `Image`, `Modal`, `Picker` и др.)

### Структура проекта

#### `ui/views/VacancyDetailView.tsx`
Основной экран деталей вакансии и отправки отклика.

Что делает:
- отображает основную информацию о вакансии;
- показывает описание, требования и условия;
- содержит кнопку **«Откликнуться»**;
- открывает модальное окно для выбора резюме;
- вызывает use case для проверки соответствия резюме требованиям вакансии.

#### `data/mocks/resumes.ts`
Слой данных с моками резюме.

Что содержит:
- тип `Resume`;
- массив `resumes`;
- флаг `isRecommended`, показывающий, считается ли резюме подходящим.

#### `domain/usecases/checkSkillsMatch.ts`
Доменный use case для проверки соответствия резюме вакансии.

Что делает:
- инкапсулирует правило проверки;
- возвращает результат проверки (`isMatch`);
- позволяет UI-слою не зависеть от деталей бизнес-логики.

### Как запустить проект

1. Клонировать репозиторий:

```bash
git clone <URL_репозитория>
cd mobile
```

2. Установить зависимости проекта:

```bash
npm install
```

3. Установить пакет для локального хранения данных:

```bash
npx expo install @react-native-async-storage/async-storage
```

4. Запустить проект в браузере:

```bash
npm run web
```

### Где находятся моковые данные

#### `data/mocks/resumes.ts`

Файл содержит тестовые резюме, которые используются на экране отклика.  
Каждое резюме описано типом `Resume` и содержит признак, определяющий, подходит ли оно для вакансии.

### Реализованный сценарий

В приложении реализован демонстрационный пользовательский сценарий: просмотр вакансии и отправка отклика с выбором резюме.

### Как проверить сценарий

1. Откройте экран деталей вакансии.
2. Убедитесь, что отображаются:
   - название вакансии;
   - компания;
   - зарплата;
   - город / формат работы;
   - разделы **«Описание»**, **«Требования»**, **«Условия»**.
3. Нажмите кнопку **«Откликнуться»**.
4. В модальном окне:
   - выберите одно из резюме;
   - подтвердите отправку отклика.
5. Проверьте два сценария:
   - если выбрано неподходящее резюме, показывается предупреждение;
   - если выбрано подходящее резюме, отображается успешная отправка отклика.

### Что можно проверить в проекте

- корректное отображение информации о вакансии;
- работу с моковыми данными;
- выбор резюме из списка;
- обработку ошибки при отсутствии выбора;
- предупреждение о несоответствии навыков;
- успешный сценарий отправки отклика.
