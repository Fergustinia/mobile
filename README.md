## JobFinder — мобильное приложение для поиска работы

JobFinder — учебное мобильное приложение, в котором соискатель может просматривать карточку вакансии и откликаться на неё, выбирая одно из своих резюме. Проект сделан в рамках командного задания («Команда №13»).

### Что это за проект

- **Цель**: продемонстрировать типичный пользовательский сценарий в приложении для поиска работы: просмотр деталей вакансии и отправка отклика с выбранным резюме.
- **Функциональность первой версии**:
  - просмотр экрана деталей вакансии;
  - выбор резюме из списка (моки);
  - попытка отклика на вакансию;
  - отображение предупреждения при неподходящем резюме (имитация проверки матчинга навыков).

### Технологический стек

- **Язык**: TypeScript
- **Фреймворк**: React Native
- **UI**: стандартные компоненты `react-native` (`View`, `Text`, `ScrollView`, `TouchableOpacity`, `Image`, `Modal`, `Picker` и др.)

> При интеграции в полноценный проект предполагается использование типичного React Native/Expo-стека (Node.js, npm/yarn, Metro bundler).

### Структура проекта (на текущем этапе)

- **`ui/views/VacancyDetailView.tsx`** — основной экран деталей вакансии и отправки отклика.
  - отображает основную информацию о вакансии (компания, зарплата, город/формат работы, описание, требования, условия);
  - содержит кнопку «Откликнуться», открывающую модальное окно;
  - внутри модального окна позволяет выбрать резюме и подтвердить отправку отклика;
  - в зависимости от выбранного резюме может показать предупреждение о несоответствии навыков.

По мере развития проекта сюда будут добавлены:

- навигация между экранами;
- отдельные модули данных (моки);
- ViewModel/UseCase для работы с данными и состояниями.

### Как запустить приложение

Так как репозиторий содержит UI-экран `VacancyDetailView`, он рассчитан на использование как часть React Native/Expo‑проекта.

1. **Клонировать репозиторий**

   ```bash
   git clone <URL_этого_репозитория>
   cd mobile
   ```

2. **Убедиться, что есть окружение React Native/Expo**

   - установлен **Node.js** (рекомендуется LTS);
   - установлен один из менеджеров пакетов: **npm** или **yarn**;
   - при использовании Expo — глобально установлен `expo-cli` или использование `npx expo`.

3. **Интегрировать экран в существующий RN/Expo‑проект**

   - перенести папку `ui` (или `ui/views/VacancyDetailView.tsx`) в папку исходников проекта (например, `src/ui/views/`);
   - подключить экран в навигацию (например, через React Navigation) и назначить его как один из экранов стека.

4. **Запустить приложение (пример для Expo)**

   ```bash
   npm install
   npm run start
   # или
   npx expo start
   ```

   Затем открыть приложение в эмуляторе или на устройстве через Expo Go.

> Если у вас уже есть готовый React Native проект, достаточно перенести `VacancyDetailView` и добавить его в текущую навигацию, после чего запуск производится стандартной для проекта командой.

### Где находятся моки данных

На текущем этапе моки данных находятся прямо в экране:

- **`ui/views/VacancyDetailView.tsx`**
  - массив `resumes` внутри компонента `VacancyDetailView` содержит список тестовых резюме:
    - «Резюме без опыта (не соответствует)»;
    - «Резюме Middle React Native»;
    - «Резюме Frontend 2025».

В следующих итерациях планируется:

- вынести моки в отдельный модуль (например, `data/resumes.ts` или `mocks/resumes.ts`);
- использовать эти моки из ViewModel/UseCase, а не из UI‑компонента напрямую.

### Реализованный сценарий и как его проверить

**Сценарий**: просмотр деталей вакансии и отклик с выбором резюме.

1. Откройте экран `VacancyDetailView` в приложении.
2. Убедитесь, что видите:
   - заголовок и базовую информацию о вакансии (компания, зарплата, город/формат работы);
   - разделы «Описание», «Требования», «Условия».
3. Нажмите кнопку **«Откликнуться»**:
   - откроется модальное окно с краткой информацией о вакансии;
   - станет доступен список резюме для выбора.
4. **Выбор резюме и отправка отклика**:
   - выберите одно из резюме из списка;
   - нажмите кнопку подтверждения отклика.
5. **Проверка поведения при «плохом» резюме**:
   - для резюме, которое помечено как «не соответствует», приложение должно показать предупреждение о несоответствии навыков;
   - для подходящего резюме модальное окно закрывается, и отображается сообщение об успешной отправке отклика.

Таким образом, в рамках одной демонстрационной ветки сценария можно проверить:

- корректность работы с моковыми данными (список резюме);
- базовую обработку ошибок (попытка отправить отклик без выбранного резюме, предупреждение о несоответствии навыков);
- нормальное состояние экрана (отображение контента вакансии и модального окна).
